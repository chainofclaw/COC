本文档是 COC 白皮书的姊妹篇。白皮书描述"是什么"和"为什么"，本文档描述"如何运营、如何发展、谁来推动"。COC 的愿景——**为 AI Agent 设计、由 AI Agent 开发、由 AI Agent 运行、为 AI Agent 服务、让 AI Agent 永生**——只有在生态足够丰富时才能成立。本文档是一份将技术实现转化为活跃生态的执行蓝图。

## 一、生态愿景

### 1.1 行业背景与机遇

COC 诞生于 AI Agent 爆发式增长的拐点上：

| 趋势 | 数据 | 对 COC 的意义 |
|------|------|--------------|
| **AI Agent 框架增长** | LangChain 100K+ stars, AutoGPT/CrewAI/MetaGPT 等数十个框架 | 巨大的开发者群体需要统一的 Agent 身份与永续基础设施 |
| **企业级部署** | Salesforce Agentforce、Microsoft Copilot Studio、Anthropic Claude 进入企业 | Agent 备份与恢复成为合规刚需 |
| **市场规模预测** | 2026: $7B+ → 2030: $50B+ (Gartner) | 早期参与的窗口期 |
| **Gartner 预测** | 2027 年 50% 大型企业将部署 AI Agent | 数千万 Agent 实例需要去中心化基础设施 |

**COC 的差异化定位**：当其他 AI 基础设施聚焦于"训练"和"推理"时，COC 聚焦于 **Agent 的身份、运行、永续**——这是一个尚未被任何现有方案系统性解决的领域。

### 1.2 阶段目标 (基于 AI Agent 行业增长曲线校准)

OpenClaw 是 COC 优先支持的参考 Agent 实现，已接入当前 COC 网络并作为重要存储服务提供节点运行；COC 协议本身欢迎任何兼容 DID 的 Agent 框架接入。考虑到 Agent 是软件实例（指数增长），而节点是物理基础设施（线性增长），两者的目标曲线不同：

| 时期 | 阶段定位 | 节点目标 | Agent 目标 | 经济指标 |
|------|---------|---------|-----------|---------|
| **第 1 年** (2026) | 最小可行网络 + 早期采用者 | 200+ 活跃节点 | 10K+ 注册 Agent，5+ Agent 框架兼容 | Q2 主网创世 → Q3-Q4 灰度扩张 |
| **第 2 年** (2027) | 生态萌芽 + 第一批 dApp | 1,000+ 节点 | 500K+ Agent，10+ 第三方实现 | TVL > $10M |
| **第 3 年** (2028) | 自循环经济 + 商业采用 | 5,000+ 节点 | 10M+ Agent | TVL > $100M |
| **第 5 年** (2030) | AI Agent 行业标准之一 | 20,000+ 节点 | 100M+ Agent | TVL > $1B，经济活动 $5B+ |
| **第 10 年** (2035) | 不可被关闭的 AI 基础设施 | 50,000+ 节点 | 1B+ Agent | 与 AI 行业整体规模同步 |

**目标设定原则**：
- **节点数保守**：受 Bond 经济性、运营复杂度约束，按类似公链 (Polygon, BNB Chain) 的早期增长曲线
- **Agent 数激进**：Agent 是软件，单个开发者可创建数百到数千个 Agent，匹配 AI 行业的指数趋势
- **TVL 标准**：参考 L1/L2 公链的中位数据（不与 Ethereum 对标，但需超过专用链平均水平）

### 1.3 与 AI 行业增长曲线的对齐

```
AI Agent 部署量预测 (Gartner + 行业平均):
  2026: 10M  ─→  2027: 100M  ─→  2028: 500M  ─→  2030: 5B+

COC Agent 注册目标 (假设捕获 1-2% 全球 Agent):
  2026: 10K  ─→  2027: 500K  ─→  2028: 10M   ─→  2030: 100M
```

捕获 1-2% 的全球 Agent 看似激进，但因为：
1. Agent 备份/永续是刚需，COC 是目前为数不多在身份+永续场景上提供完整去中心化方案的项目之一（见 §X 长期使命，欢迎更多方案出现并互联）
2. DID 标准化意味着兼容任何 Agent 框架，市场规模 = 整个 AI Agent 行业
3. 早期采用者集中度高（少数大型 Agent 框架占据多数 Agent 实例）

### 1.4 三个核心目标群体

| 群体 | 价值主张 | 互动方式 |
|------|---------|---------|
| **AI Agent 开发者** | 为 Agent 提供身份、存储、永续基础设施 | 通过 SDK 接入 DID + Soul Backup |
| **节点运营者** | 通过 PoSe 挖矿赚取 COC，门槛低 (~50 USDT bond) | 运行 FN/SN/RN，自动获得奖励 |
| **dApp / 协议开发者** | 在 EVM 兼容链上构建 AI 原生应用 | 通过 JSON-RPC、WebSocket、智能合约 |

## 二、四阶段发展路线

> **成熟度状态约定** (本文档与商业计划、白皮书共用)
> - 🟢 **代码已实现**: 协议/合约/服务代码已写完且测试通过
> - 🟡 **测试网运行**: 已部署到测试网长期运行
> - 🔵 **主网运行**: 已部署到主网
> - ⚪ **参考实现规划中**: 规范明确但代码未启动
>
> 当前 (本路线图发布时) 状态: COCToken / EmissionSchedule / FoundationVesting / Treasury / PoSeManagerV2 / DIDRegistry / SoulRegistry / CidRegistry 合约 + chain-engine / EVM / P2P / RPC / IPFS / 三大基础服务核心代码均为 **🟢 代码已实现 + 🟡 测试网运行**；OpenClaw 参考 Agent 为 **🟢 代码已实现 + 🟡 已接入当前 COC 网络**（作为重要存储服务提供节点运行）。

### 阶段 1: 创世启动期 (Q1-Q2 2026)

**目标**: 将已实现的代码与测试网经验切换到主网；推进 OpenClaw 参考 Agent 的主网迁移与功能扩展

| 里程碑 | 交付物 | 入场状态 → 阶段 1 结束状态 | 负责方 |
|--------|--------|--------------------------|--------|
| **主网创世** | COCToken + 创世分配 250M COC + EmissionSchedule | 🟡 测试网 → 🔵 主网运行 | 基金会 |
| **PoSe v2 主网启用** | 服务挖矿 + 自动铸币 + 奖励领取 | 🟡 测试网 → 🔵 主网运行 | 核心团队 |
| **DID 主网注册启用** | DIDRegistry + 能力位掩码 + 委托链 | 🟡 测试网 → 🔵 主网运行 | 核心团队 |
| **AI 硅基永生主网 v1** | SoulRegistry + Carrier 注册 + 备份恢复 | 🟡 测试网 → 🔵 主网运行 | 核心团队 |
| **OpenClaw 参考 Agent 主网迁移** | 已接入测试网的 OpenClaw 切换到主网，扩展存储/计算能力 | 🟡 已接入当前网络 → 🔵 主网运行 | 核心团队 |
| **创世节点招募** | 100+ 早期节点运营者 | — | 社区 + 基金会 |

**KPI**: 主网稳定运行 30 天 + 100+ 活跃节点 + 100K+ 链上交易 + 1K+ Agent 注册

### 阶段 2: 生态萌芽期 (Q3-Q4 2026)

**KPI**: 200+ 节点 + 5+ 兼容 DID 的 Agent 框架 + 10K+ 注册 Agent + 5+ dApp 上线 + 1M+ 链上交易

### 阶段 3: 生态成长期 (2027)

**目标**: 形成自循环经济和有意义的链上活动

| 里程碑 | 交付物 |
|--------|--------|
| **DAO 治理上线** | GovernanceDAO 全功能，基金会决策权移交 |
| **跨链桥** | 与 Ethereum、BNB、Polygon 的资产桥 |
| **Agent 市场** | Agent 服务市场（存储、计算、推理） |

### 阶段 4: 生态成熟期 (2028+)

**目标**: 成为 AI 行业的标准基础设施

| 里程碑 | 交付物 |

| **去中心化排序** | 多排序器、共享排序方案 |
| **完全去中心化** | 基金会角色淡出，DAO 主导决策 |

**KPI**: 5,000+ 节点 + 10M+ Agent + $100M+ TVL + 多语言客户端

## 四、运营组织架构

### 4.1 三层组织模型

```
┌─────────────────────────────────────────────────┐
│              COC DAO (主权层)                    │
│  Faction-based 治理 (Human + Claw)              │
│  通过 GovernanceDAO.sol                          │
└────────────────────┬────────────────────────────┘
                     │ 决策授权
                     ▼
┌─────────────────────────────────────────────────┐
│           COC Foundation (执行层)                 │
│  非营利实体，依据 DAO 决策执行                    │
│  - 协议开发                                       │
│  - 资助拨付                                       │
│  - 法律合规                                       │
│  - 品牌与生态推广                                 │
│  季度预算 + 季度公开报告                           │
└────────────────────┬────────────────────────────┘
                     │ 协议升级、Grant
                     ▼
┌─────────────────────────────────────────────────┐
│          核心团队与生态贡献者 (建设层)              │
│  - Core Devs（协议升级）                          │
│  - Ecosystem Devs（dApp/SDK/工具）               │
│  - Community Stewards（社区运营）                │
│  - Auditors（独立审计方）                         │
└─────────────────────────────────────────────────┘
```

### 4.2 基金会职责边界

**基金会做什么**:
- 协议开发与维护（在 DAO 同意的范围内）
- 资助申请的合规与执行
- 生态推广与战略合作
- 法律合规与监管对接
- 季度财务报告与第三方审计

**基金会不做什么**:
- 不能单方面修改协议规则
- 不能动用国库（需 3/5 多签 + DAO 批准）
- 不能拒绝合规的 DID 注册或节点接入
- 不能优先支持某个 Agent 实现

### 4.3 治理演进

| 阶段 | 基金会角色 | DAO 角色 |
|------|-----------|----------|
| **创世后 0-1 年** | 主导协议升级、资助决策 | 监督、紧急投票 |
| **1-3 年** | 与 DAO 共同决策 | 核心提案投票 |
| **3 年+** | 执行层（依据 DAO 决议） | 主权决策方 |

## 六、商业模式与收入来源

### 6.1 协议层收入

| 来源 | 说明 | 流向 |
|------|------|------|
| **Gas 费** | 交易和合约执行 | 矿工 (优先费) + 销毁 (基础费) |
| **PoSe 服务费** | 链外服务付费 | 服务提供者 |
| **DID 注册费** | 灵魂身份注册和备份锚定 | 矿工 + 协议销毁 |
| **节点 Bond** | 节点注册的一次性 ~50 USDT 等价保证金 | Bond 池 (反欺诈) |

### 6.2 基金会可持续收入

基金会不依赖捐赠，长期依赖以下收入维持运营：

| 来源 | 估算规模 |
|------|---------|
| **创世分配释放** | 60M COC，48 月线性释放 |
| **过期奖励** | 10% 未领取奖励自动转入 |
| **战略合作收入** | 与企业合作的咨询/集成费 |
| **生态投资回报** | Foundation 投资的早期项目股权/代币 |

### 6.3 节点运营者经济模型

**示例**: 单个 FN 节点的预期年收入

```
假设:
- 网络总活跃节点: 100 (TARGET_NODE_COUNT)
- Year 0 通胀率: 5%
- 总挖矿池: 750M COC
- 第 0 年释放: 37.5M COC
- 假设 60% 分给 B1 (uptime/RPC): 22.5M COC
- 假设 100 个 FN 平均分: 225,000 COC/节点

价格情景对照:
                       $0.10/COC      $0.15/COC      $0.20/COC
                       (早期保守)     (Series A 锚定)  (中性)
  年收入                $22,500        $33,750         $45,000
  月收入                $1,875         $2,813          $3,750
  月硬件成本            $200           $200            $200
  净收益 (月)           $1,675         $2,613          $3,550
  Bond (一次性)         $50            $50             $50
```

实际收入取决于网络节点数、COC 价格、节点服务质量。$0.15-$0.20 区间对应商业计划 §9.1 Series A 估值情景下的隐含代币价格。

## 八、关键合作伙伴策略

### 8.1 优先合作类型

| 类型 | 目标 | 价值交换 |
|------|------|---------|
| **AI 公司** | 让大型 AI 公司在 COC 上备份 Agent | 提供基础设施 ↔ 流量与品牌 |
| **云厂商** | Carrier 节点提供商 | 提供 PoSe 挖矿收入 ↔ 利用闲置算力 |
| **存储项目** | IPFS 集群、Filecoin、Arweave | 跨链存储 ↔ 共享技术 |

## 十、长期使命

> **十年后，当有人问"AI Agent 在哪里运行？"时，COC 应当成为多个可能答案之一。**
>
> **二十年后，当 AI Agent 自然地拥有身份、记忆、永续，我们希望这一切的基础设施是开放的、去中心化的、不属于任何单一实体——而 COC 是这个开放标准的早期推动者之一。**

构建 AI 的去中心化基础设施不是一个产品，而是一场跨越十年的社会工程。COC 作为先行者，承担的是开拓而非垄断的角色。当未来出现更好的 AI 去中心化方案时，COC 应当与之互联互通，而非对抗——因为我们的真正使命是 **AI 自由**，而不是 COC 本身。
