## 全局设计要求

### 视觉风格
- **主色**: 深蓝 (#0A1628) + 强调色琥珀金 (#F5A623) + 辅助色冷白 (#F8F9FB)
- **字体**: 标题使用无衬线粗体 (如 PingFang SC Bold / Inter Bold)；正文使用 PingFang SC Regular / Inter Regular
- **风格关键词**: 极简、克制、技术感、投资级专业度（避免炫技动画）
- **建议比例**: 16:9 横向
- **页边距**: 上下各 60px，左右各 80px

### 通用元素
- **页眉**: 左上角小尺寸 COC logo + 右上角章节序号 + 当前页码 / 总页数
- **页脚**: "COC | clawchain.io | 2026-04-07 | Confidential"
- **章节纵览页**: 占满全屏的章节标题 + 一句话摘要 + 该章节将回答的 2-3 个核心问题
- **细节页**: 标题区 (顶部 15%) + 主内容区 (中部 70%) + 关键 takeaway 区 (底部 15%)

### 数字与图表规范
- **关键数字**: 用强调色 (#F5A623)、加粗、字号比正文大 1.5-2 倍
- **图表色板**: 深蓝主色 + 琥珀金强调 + 灰度对比；避免彩虹色
- **数据来源**: 任何引用的行业数据 (Gartner / L2Beat / Chainspect 等) 必须在右下角小字标注来源
- **状态标签**: 沿用 BP 的四档约定 🟢 / 🟡 / 🔵 / ⚪，颜色与表情符号一致

### 演讲者备注约定
每页底部以 `🎤 演讲者备注` 标注口头讲解要点，30-90 秒一页（总演讲时长 ~30-40 分钟）。

### Page 11 — P2P 文件存储 (服务 ① 详解)

**设计要求**:
- 顶部一句话定义
- 中间放一个数据流图：Agent → 上传 → IPFS → PoSe v2 验证 → Merkle 证明 → 链上锚定
- 右侧列出三个差异化点

**页面文字**:
```
P2P 文件存储 — 详解

定义：为 AI Agent 提供去中心化、抗审查、可验证的数据持久层

数据流：
Agent → AES-256-GCM 加密 → IPFS 内容寻址 → PoSe v2 服务证明 →
Merkle 树根 + CID 锚定到 SoulRegistry → 链上 EIP-712 签名

差异化：
✓ 增量备份：只存变更文件，通过 parentCid 链接历史
✓ PoSe 验证：服务节点的存储证明可被链上挑战
✓ 成本可控：高 TPS + Rollup → 微支付计费可行

成熟度：🟢 + 🟡（详见 §3.1）
```

**🎤 演讲者备注**:
存储在 Web3 不是新东西，但**专为 Agent 设计的存储 + 服务证明 + 链上锚定一体化**是新的。我们用 PoSe v2 把服务交付转成链上可验证事件，这是很多 IPFS 项目缺失的环节。

去中心化身份 (DID) — 详解

定义：W3C 标准 did:coc 方法，为 AI Agent 提供主权身份

身份层次：
- DID Identifier (did:coc:abc123...)
- 能力位掩码 (16 bit, 12 位已定义 + 4 位预留)
   ┌─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┐
   │S│C│V│Ch│A│W│R│B│G│I│D│F│ │ │ │ │

DID 是 COC 最重要的差异化原语之一。**它不只是"链上账户"——它是 Agent 的法律身份、能力清单、授权链**。Aave 类协议未来可以基于 DID 信誉给 Agent 差异化利率，而不是匿名地址。

**两个边界请特别注意**：(1) 治理参与权默认**没有特殊否决权**，主权决策由 DAO 规则决定；(2) 董事会默认是**观察员席位**，正式表决董事席位需要单独法律审批。这与 §IV 的非营利基金会定位一致，避免任何"购买治理控制权"的合规误读。
